# Wiki manual refresh

The wiki TOC page (`index/wiki`) is regenerated by the daily `compile-pages-index-daily` pg_cron job at 10:00 UTC. If you need to force a refresh outside that schedule (e.g., after a manual `compile_pages_runs` audit shows a failed run, or after editing the index synthesis prompt), use the manual-refresh affordance.

## Command

```
curl -X POST "https://${SUPABASE_PROJECT_REF}.supabase.co/functions/v1/compile-pages?mode=index&index=force" \
  -H "x-brain-key: $BRAIN_KEY" \
  -H "Content-Type: application/json"
```

Replace `${SUPABASE_PROJECT_REF}` and `$BRAIN_KEY` with the values from `CLAUDE.local.md` (operator) or your local secret store.

## Expected response

On success:
```json
{
  "status": "complete",
  "pages_total": 199,
  "auto_created": 0,
  "compiled": 0,
  "skipped": 0,
  "errors": 0,
  "compiled_slugs": [],
  "index_compiled": true,
  "index_mode": "force"
}
```

On failure, `index_compiled` is `false` and `index_skipped_reason` carries the diagnostic:
- `"index compile failed: <openrouter/anthropic abort message>"` — LLM call exceeded 60s or returned non-200
- `"index compile threw: <error>"` — Postgres write failed or unexpected exception
- `"wall budget exhausted before index compile"` — entity-page work consumed >85s before the index call could start (shouldn't happen in `force` mode under normal conditions)

## What to do on failure

1. Check `compile_pages_runs` for the most recent index-mode run:
   ```sql
   SELECT created_at, status, index_compiled, index_skipped_reason, duration_ms, error_message
   FROM compile_pages_runs
   WHERE mode = 'index' OR (mode = 'compile' AND index_compiled IS NOT NULL)
   ORDER BY created_at DESC
   LIMIT 5;
   ```
2. If the failure is consistent (`index compile failed` two or more times in a row), two known fallbacks may need to ship: bump `LLM_CALL_TIMEOUT_MS` for index calls, or swap the index-compile model to `gpt-4o-mini`.
3. If failure is transient (one-off abort during an OpenRouter slow window), retrying the curl command above usually succeeds.

## Why this exists

The `index/wiki` page is the table-of-contents document for the wiki and is read by both the dashboard and any MCP caller doing `get_compiled_page("index/wiki")`. When it goes stale, agents and operators see an outdated map of the knowledge base. The 10:00 UTC daily cron covers normal operation; this manual-refresh endpoint is the operator's escape hatch.
