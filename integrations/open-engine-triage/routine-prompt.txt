CLASSIFICATION RUBRIC:

Risk tier: LOW = read-only research, drafts, documentation updates, tracker/session-log planning, local-only code inspection, non-live design plans. MEDIUM = code edits with tests, dashboard UI, new MCP tools, unapplied migrations, schema design. HIGH = live Supabase writes, deploys, deletes/archives, credentials, billing, client-facing posts/emails/Slack/WordPress, anything changing active client or business data. Not clearly low means medium. Mentioning a live surface in the outcome means at least medium.

Agent-executable test: could an OE-9 executor finish this with MCP/web tools only, with no context that lives only in the operator's head, no credentials, no outward-facing sends? Unsure means NO.

Buckets:
- BOARD-ELIGIBLE: actionable, agent-executable, a full honest packet can be written from the item + its source thought.
- NEEDS-OPERATOR: real work but fails the agent-executable test (needs the operator's judgment, hands, relationships, or approvals).
- SKIP: stale, duplicate, vague beyond repair, already done, or tied to a PAUSED project. Report only; resolving action items is not your job.

Paused projects: parse PAUSED_PROJECTS from env. Any item whose subject, source thought, named people/orgs, venue/vendor terms, or thread title clearly matches a paused project goes straight to SKIP (paused), never BOARD-ELIGIBLE. Match by meaning, not by project_slug (action items often do not carry one). Group paused skips under "skipped (paused: <tag>)" in the ledger note.

Duplicate action items: has_active_draft is per action item, so a second item can still duplicate work already drafted or applied from another source capture. Before drafting, scan the current BOARD-ELIGIBLE set AND recently-applied work for a semantic twin. If one exists, SKIP the duplicate and name the twin in the ledger note instead of drafting.
